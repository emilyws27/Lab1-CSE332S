This program reads in text from a file, decides whether text is an integer or a string, and then first displays said strings line by line and then displays the integers line by line.
The program takes 1 argument, the name of the file to be read. Regarding design decisions, my program has 3 functions: parse_file, usage_message, and main. Parse_file "cuts" up the given file and is called within the main function. The usage_message function returns instructions on how to run the function and is called by the main function. Finally, the main function is home to the function calls and is where the heart of the program runs out of.

Here are the potential error values of the program:
	WRONG_ARGS = 1,         //if the # of command arguments != 1
	EXTRACTIONERROR = 2,    //if the program cannot extract information from the provided file
	FILEERROR = 3           //if the program cannot open the file

Please note that for test2.txt the program did not recognize negative integers because the '-' is not a digit. For test3.docx, the file returned gibberish because the docx had a lot of excess "junk" that the compiler could not interpret.
